# 文件模板

首次搭建学习文档体系时，用以下模板创建全套文件。`[占位符]` 处替换为用户实际信息。

---

## index.md

```markdown
# [用户名]的编程学习手册

> **背景：** [如：产品经理 / 商科背景，代码零基础]
>
> **目标：** [如：理解技术底层逻辑，能独立用 AI 把项目做出来]
>
> **工具：** [如：Cursor + AI（Ask 模式研究，Agent 模式执行）]
>
> **文档原则：** 逻辑清晰、好理解、照着做就行、不务虚。

---

## 文件导航

| 文件 | 干什么用 | 什么时候翻 |
|------|---------|-----------|
| [playbook.md](playbook.md) | SOP + 提示词 + 命令速查 | 做事时照着走 |
| [knowledge.md](knowledge.md) | 概念按主题沉淀 | 碰到不懂的词回来查 |
| [learning-log.md](learning-log.md) | 每日学习速记 | 每天学完速记 |
| [open-questions.md](open-questions.md) | 还没搞懂的问题 | 学习中随时记 |
| [projects/](projects/) | 项目研究笔记 | 研究新项目时 |

---

*本文件是导航索引。具体内容在上方各文件中。*
```

## playbook.md

```markdown
# 实操手册

> 做事时翻这个。SOP + 提示词 + 命令，照着做就行。

---

# 一、SOP 区

---

## SOP #1：研究别人的项目

### 适用场景

- 拿到一个别人的项目想搞懂它
- 调研技术方案 / 学习架构思路

### Step 1：判断项目类型

> 拿到项目第一件事：扫文件列表，判断类型。有 README.md 就先读它。

| 关键标志 | → 类型 | 怎么用 |
|---------|--------|-------|
| 有 `SKILL.md` | Cursor Skill | 放到 Skill 目录，对话触发 |
| 有 `package.json` / `requirements.txt` | 程序/应用 | 装依赖 → 启动 |
| 只有一两个 `.py` / `.sh` 文件 | 脚本 | 看注释找用法，终端直接跑 |
| 主要是 `.md` 文件 | 纯文档 | 打开阅读 |

### Step 2：环境准备（程序类才需要）

1. 创建独立环境（不用 base）
2. 安装依赖
3. 配置密钥（`.env`）
4. 建 Git 基线（`git init` + 首次 commit）
5. 运行，确认能跑

### Step 3：理解项目（三轮递进）

**第 1 轮：**

> "用小白能听懂的方式帮我分析这个项目。"

**第 2 轮：**

> "帮我整理出这个项目的 spec：文件职责表、流程图、核心规则、关键设计决策。不用逐行细节。"

**第 3 轮：** 针对不懂的部分逐个深挖

> "我不太理解 XXX 部分，展开讲讲。"

### 速查卡片

| 步骤 | 做什么 | 模式 |
|------|-------|------|
| Step 1 | 扫文件列表 → 判断类型 | 肉眼 |
| Step 2 | 建环境 → 装依赖 → 跑起来 | 终端 |
| Step 3 · 1 轮 | "帮我分析这个项目" | Ask |
| Step 3 · 2 轮 | "整理 spec" | Ask |
| Step 3 · 3 轮 | "XXX 展开讲讲" | Ask |

---

## Python 环境管理

```bash
conda create -n 项目名 python=3.12      # 创建独立环境
conda activate 项目名                    # 激活
conda env list                          # 查看所有环境（安全只读）
pip install -r requirements.txt         # 安装依赖
```

## Git 基础

```bash
git init && git add . && git commit -m "基线"    # 首次建仓库
git add .                               # 暂存改动
git commit -m "说明"                     # 本地提交
git push                                # 推到远端
git restore .                           # 丢弃未提交修改（危险）
git log --oneline                       # 查看历史（安全只读）
```

## knowledge.md

```markdown
# 知识库

> 学到的概念在这里沉淀。按主题分区，新概念加到对应区块下方。

---

## 概念组 A：代码怎么组织和运行

**代码需要"家"（仓库）、"版本"（Git）、"食材"（依赖）和"土壤"（环境）。**

| 概念 | 一句话定义 | 生活类比 |
|------|----------|---------|
| **Repo** | 一个项目的所有代码和历史 | GitHub 上的一个项目 |
| **Git** | 代码版本管理工具 | 文档的"历史版本" |
| **Dependencies** | 别人写好的工具包 | 做菜的"预制调料包" |
| **`.env`** | 存密码密钥的文件 | 保险箱——绝不提交到 Git |

---

*(学到新概念，找对应主题区加一行。没有对应区就新建一个。)*
```
