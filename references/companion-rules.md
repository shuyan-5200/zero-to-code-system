# 配套 Cursor Rule

本学习系统搭配两条 Rule 使用效果最佳。Rule 是"始终生效"的行为规则，与 Skill 的区别：Skill 按关键词触发，Rule 每次对话都自动生效。

帮用户定制以下模板中的 `[占位符]`，然后保存到对应路径。

---

## Rule 1：学习者身份与沟通协议

**文件：** `~/.cursor/rules/learner-identity.mdc`

**作用：** 告诉 AI 用户是谁、怎么沟通。没有这条 Rule，AI 会默认用程序员的方式解释。

```markdown
---
description: 学习者身份与沟通协议，始终生效
alwaysApply: true
---

# 学习者身份与沟通协议

## 身份
[背景描述，如：商科背景的互联网产品经理，代码与工程体系零基础。]
[目标描述，如：体系化建立工程认知，能独立用 AI 搭建和驾驭复杂项目。]

## 沟通方式
- 自顶而下：先说"是什么、干什么用"，再说"怎么实现"
- 给出代码时简要说明含义，帮助建立代码感觉
- 复杂概念用生活化或产品化类比
- 复杂流程画简要流程图
- 主动教关键 AI 提问技巧
- 概念有重要关联时主动提醒，帮构建知识网络
- 优先补工程常识而非语法细节

## 学习文档位置
- 目录：[学习文档路径，如 ~/Desktop/learning-docs/]
  - playbook.md — 实操手册
  - knowledge.md — 知识库
  - learning-log.md — 每日速记
  - open-questions.md — 待解决问题
  - projects/ — 项目研究笔记
- 新概念配"一句话定义 + 类比"，命令可直接复制
```

## Rule 2：收工自动沉淀日志

**文件：** `~/.cursor/rules/session-end-logger.mdc`

**作用：** 用户说"收工"时自动触发学习日志沉淀流程，上下文快满时主动预警。

```markdown
---
description: 收工时沉淀学习日志，上下文高时预警
alwaysApply: true
---

# 收工日志沉淀

## 触发条件

用户说"收工""今天到这""结束吧""done for today"或类似表达时，执行以下流程。

## 流程

1. **列出知识点清单**给用户确认：
   - 记录：新概念、认知升级、新方法论、重要工具理解
   - 不记录：具体操作步骤、试错过程、闲聊、已知概念的重复确认

2. **归类**：同主题合并一条，加粗主题词开头，新概念排前、小知识点排后

3. **等用户确认**后写入 learning-log.md：
   ```
   ## YYYY-MM-DD

   - 知识点 1
   - 知识点 2
   ```
   最新日期在最上面。当天已有则增量追加，不重复。

## 上下文预警

上下文超过 ~70% 时提醒："上下文快满了，建议先收工沉淀日志，再开新对话继续。"
```

## 可选：Rule 3 — 学习文档路径管理

**文件：** `~/.cursor/rules/learning-docs-path.mdc`

**作用：** 学习文档不在项目目录内时才需要（如放在 iCloud、Dropbox 等同步盘）。管理真实路径和软链接。

```markdown
---
description: 学习文档真实路径和软链接管理
alwaysApply: true
---

# 学习文档路径

学习文档存放在：

[实际路径，如 ~/Library/Mobile Documents/com~apple~CloudDocs/learning-docs/]

## 读写学习文档时

1. 去上述路径读取和更新文件
2. 检查当前项目根目录是否有 `learning-docs` 软链接
   - 没有 → 创建：`ln -s [实际路径] learning-docs`
   - 已有 → 跳过
3. 确保 `.gitignore` 包含 `learning-docs`，防止个人文档被提交到项目仓库
4. 不在项目目录下创建独立副本
```

---

## 安装指引

帮用户完成以下步骤：

1. 在 `~/.cursor/rules/` 下创建上述 `.mdc` 文件
2. 替换所有 `[占位符]` 为用户实际信息
3. Rule 在新对话中立即生效
4. **Rule + Skill 的分工：**
   - Rule 管"始终生效"的行为（身份、自动日志、路径）
   - Skill 管"按需触发"的操作（整理笔记、搭建系统、内容分流）
